Projects

To the New World

My first personal project. "To the New World" (TNW) has a backend server that generates new data to feed to a "world" (PostGreSQL Database) which evolves in real time on the front end. Its artistic vision was to capture the cyclical yet unpredictable nature of history. This project also marks my first foray into external libraries, as the CreateJS library was used for all graphics work done in the HTML5 canvas elements. It was deployed to Heroku in July 2016.

Brown Maps

A school project that uses java to interpret a sqlite database of points and roads. This "map" is then analyzed to find optimal paths between points using Dijkstra's algorithm. Front end features include map visualization, autocorrection, click input, and highlighted roads. The project also ran with a "traffic server" that would generate different traffic values for each road, which would then change color on the front end accordingly.

Agent 32

This school project uses the Google Maps API to put a new spin on the classic battleship game. Agent 32 uses Java's networking features for player lobby systems and chat. It uses the Maps API to generate "radar hints" which helps players locate their opponent's "spies" on real world maps. The game also features a fully designed front end implemented with HTML, CSS, and JS.

Rubber MALLET

Created in just under 18 hours at the Bridgehacs hackthon, Rubber MALLET is by no means a lightweight app. It implements features from the MALLET machine learning package to analyze txt files for "topics" that capture the text's most important themes. It also features a simple yet distinctive GUI that uses D3 to create a "bubble" visualization of the topics.
